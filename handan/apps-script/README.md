# 判断データベース — Apps Script（データ中継API）デプロイ手順

「このアプリは Google で確認されていません」という警告を無くすため、スプレッドシートの
読み取りを **Apps Script ウェブアプリ**に移しました。ブラウザ側は「Google でログイン」の
**ID 確認だけ**を行い、機密スコープ（spreadsheets）を一切要求しません。

この API を一度デプロイし、発行された URL を `handan/config.js` の
`APPS_SCRIPT_URL` に貼れば設定完了です。

## 手順

1. https://script.google.com/ を開き、**新しいプロジェクト**を作成する。
2. 既定の `Code.gs` の中身を、このフォルダの [`Code.gs`](./Code.gs) の内容で**丸ごと置き換える**。
   - 冒頭の `CLIENT_ID` / `ALLOWED_DOMAIN` / `SHEET_ID` が正しいか確認する
     （初期値はこのリポジトリの設定と一致済み）。
3. 右上の **デプロイ** →「**新しいデプロイ**」。
   - 種類：**ウェブアプリ**
   - 説明：任意（例：判断DB API）
   - **次のユーザーとして実行**：**自分**（＝ミラー所有者の権限で読む）
   - **アクセスできるユーザー**：**全員**
     （※ ここは「誰でも API を呼べる」の意味。実際のアクセス制御は Code.gs 内で
       ID token 検証＋@seichiku.org 限定により行うので安全です。）
4. 初回は権限承認を求められる → 自分のアカウントで許可する。
   - ここで自分自身が「未確認アプリ」警告を見ることがありますが、これは**あなた（開発者）
     が一度承認するだけ**で、スタッフには表示されません。「詳細」→「移動」で進めてください。
5. 発行された **ウェブアプリ URL**（`https://script.google.com/macros/s/XXXX/exec`）をコピー。
6. `handan/config.js` の `APPS_SCRIPT_URL` にその URL を貼り付けて commit / push。

## 動作確認

- ブラウザで `…/exec` を直接開くと `{"ok":true,"msg":"…"}` が表示されれば公開OK。
- サイト（`/handan/`）で @seichiku.org アカウントのログインボタンを押し、
  警告画面が出ずにデータが表示されれば成功です。

## コードを更新したとき

`Code.gs` を編集したら、**デプロイ →「デプロイを管理」→ 既存のデプロイを編集 →
バージョンを「新規」にして保存**してください（URL は変わりません）。
新しいデプロイを作ると URL が変わるので注意。

## セキュリティ上の注意

- ブラウザは機密スコープを一切要求しないため、未確認アプリ警告は出ません。
- ドメイン判定はサーバー側（Apps Script）で ID token を検証したうえで行うため、
  クライアント側の改ざんでは突破できません。
- ミラー SS は各ユーザーへ共有する必要はありません（デプロイ主の権限で読むため）。
